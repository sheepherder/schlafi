# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Schlummershuffle (Schlafi) — a web app implementing the Cognitive Shuffle sleep technique. Users listen to random German words and visualize them to fall asleep. Audio comes from pre-rendered Google Cloud TTS files (Chirp 3: HD) with browser TTS as fallback. See `RESEARCH.md` for scientific background and feature roadmap.

## Development

No build system, no dependencies, no package manager. Open `index.html` directly in a browser to develop and test. Deployment is automated to GitHub Pages on push to `main` via `.github/workflows/pages.yml`.

## Architecture

Four source files:
- `index.html` — markup only, links to external CSS/JS.
- `style.css` — all styles (dark theme, animations, responsive layout, accessibility focus styles).
- `app.js` — all application logic (voice playback, timer, session, word selection, Wake Lock).
- `words.js` — frequency-sorted flat array of `[genus, noun]` pairs (~3500 words). Generated by `scripts/build_wordlist.py`.

**Audio generation** (`scripts/generate_audio.py`):
- Renders each word as individual `.opus` file via Google Cloud TTS (Chirp 3: HD, `v1beta1` API).
- Uses SSML with X-SAMPA phonemes (`<phoneme alphabet="x-sampa">`) to force correct "ein"/"eine" pronunciation (prevents colloquial elision like "n'").
- Leading `. ` in SSML for natural pause, trailing `.` for declarative intonation.
- ffmpeg post-processing: silence trimming (`silenceremove`, -50dB threshold), 50ms pad, Opus encoding (48kbps, `audio` application mode).
- Voices (best first): Enceladus, Sadachbia, Sadaltager, Charon. One folder per voice under `audio/<name>/`.
- Filenames: `ein_schmetterling.opus` (article + noun, lowercase, underscores). **Keep in sync with `app.js:audioFilename()`.**
- Retry with exponential backoff on 429 (rate limit: 200 req/min for Chirp 3 HD).
- Skips existing files for incremental generation.
- API key in `.env` as `GOOGLE_TTS_API_KEY`.

**Word list generation** (`scripts/build_wordlist.py`):
- Combines three data sources: word frequency (OpenSubtitles via hermitdave/FrequencyWords), noun identification + genus (Wiktionary via gambolputty/german-nouns), psycholinguistic ratings (German Psycholinguistic Toolbox, osf.io/ghjd2).
- Filters: concreteness >= 5.8, valence >= 3.5, arousal <= 4.0 (scale 1-7). Source data downloaded to `scripts/data/` (gitignored).
- Additional filters: uncountable/mass nouns (explicit list + suffix-based compound detection), medical/anatomical terms, proper nouns (two-pass Wiktionary cross-reference), duplicates (spelling variants + concept duplicates), abstract concepts, brand names, archaic words.
- Genus fixes: keeps first Wiktionary entry (most common meaning), explicit overrides for known errors (Foto→n, Tram→f, Kiwi→f).

**Voice system in app.js:**
- Pre-rendered voices auto-detected by probing `audio/<voice>/<first_word>.opus` via `new Audio()`.
- Voice dropdown: pre-rendered voices first, then browser TTS voices (with "(lokal)" label).
- Pre-rendered: plays individual `.opus` files via `new Audio(url)`. Rate/pitch sliders hidden (not applicable).
- TTS fallback: Web Speech API with German voice discovery, Chromium `synth.resume()` workaround.
- Graceful degradation: if neither pre-rendered nor TTS available, start button is disabled.

**Key browser APIs:**
- **Web Speech API** — TTS fallback for speaking words. German voices discovered dynamically via `speechSynthesis.getVoices()`. Includes a Chromium workaround (periodic `synth.resume()` every 5s to prevent pause bugs after ~15s).
- **Wake Lock API** — keeps screen on during sessions (graceful fallback if unsupported).

**Timer:** deadline-based using `Date.now()` to prevent drift during background throttling or CPU sleep.

**Word selection algorithm** (`pickWord()`): picks a random word from the flat list, avoids words from the last 50 spoken, avoids same starting letter as previous word, with progressive fallback if the pool is exhausted. Genus is mapped to indefinite article at runtime (`m/n` → "ein", `f` → "eine"). Words are spoken and displayed as "ein Haus", "eine Tür" etc.

**State management:** global variables (`running`, `timerMinutes`, `intervalSeconds`, `volume`, `speechRate`, `speechPitch`, `selectedVoice`, etc.) — no framework.

**UI pattern:** Single-page layout — settings are directly visible on the start screen (no hidden drawers or sheets). CSS-driven state toggle via `.container.running` hides settings and shows word display + timer ring. Chip selectors use `data-value` attributes and `aria-pressed` for timer/interval values.

**Visual design:** Warm dark palette (amber `#D4A46A` on near-black `#080810`) for nighttime readability. Typography: Fraunces serif for word display and title, Outfit sans-serif for UI. Subtle ambient glow (`body::before` radial gradient), film grain texture (`body::after` SVG noise), staggered `fade-up` entrance animations on settings groups. Moon crescent icon in start button.

**Accessibility:** `aria-live` on word display, `aria-pressed` on chip buttons, `role="group"` with `aria-labelledby` on chip groups, `for`/`id` on labels, `:focus-visible` styles on all interactive elements.

## Word list guidelines

Words in `words.js` are generated by `scripts/build_wordlist.py` from corpus data, filtered by:
- **Concreteness** >= 5.8 (easily visualizable, scale 1-7)
- **Valence** >= 3.5 (not negative/threatening, scale 1-7)
- **Arousal** <= 4.0 (calming, not activating, scale 1-7)
- Only pure nouns (`pos == "Substantiv"` in Wiktionary, no toponyms, affixes, etc.)
- Excludes nominalized infinitives (neuter nouns ending in -en like "das Gehen")
- Excludes false nouns (words primarily used as adverbs/adjectives/verbs)
- Excludes uncountable/mass nouns where "ein/eine" sounds unnatural (Stoffnamen, Materialien, Gewürze, etc.)
- Excludes medical/anatomical terms, proper nouns, brand names, duplicates
- Sorted by frequency in OpenSubtitles corpus, top ~3500 taken
